Generate Settings File

This page generates a settings file that can be used with a script to deploy ViPR Controller on a virtual machine (VM). After entering the information, click Generate Settings. Save "deployment.settings" file generated. After download you can customize the file name.You must generate a separate settings file for each VM (node) being deployed.

When running the script on each node, use the -file option and provide the path to the .settings file for that node. For instructions and examples on deploying a VM with this setting file, see the ViPR Controller Installation, Upgrade, and Maintenance Guide which is available from the "ViPR Controller Production Documentation Index"

Note: Mandatory to use either IPv4 or IPv6 or Both. Do not leave both with default values.

Settings for IPv4

If you are configuring an IPv4 network, each ViPR Controller node requires a unique, static IPv4 address in the subnet defined by the netmask.
If you are not configuring settings for an IPv4 network, leave the default values of 0.0.0.0 which disables IPv4.

The IPv4 address that UI and REST clients use to access ViPR. Must be different from IPv4 addresses of ViPR Servers.(The value 0.0.0.0 disables IPv4.)
IPv4 address of ViPR Server 1.(The value 0.0.0.0 disables IPv4.)
IPv4 address of ViPR Server 2.(The value 0.0.0.0 disables IPv4.)
IPv4 address of ViPR Server 3.(The value 0.0.0.0 disables IPv4.)
IPv4 address of ViPR Server 4.(The value 0.0.0.0 disables IPv4.)
IPv4 address of ViPR Server 5.(The value 0.0.0.0 disables IPv4.)
IP address of default IPv4 Gateway.(The value 0.0.0.0 disables IPv4.)
